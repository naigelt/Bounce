\doxysubsection{Detailed Description}
\doxylink{classsf_1_1_drawable}{Drawable} representation of a texture, with its own transformations, color, etc. 

\doxylink{classsf_1_1_sprite}{sf\+::\+Sprite} is a drawable class that allows to easily display a texture (or a part of it) on a render target.

It inherits all the functions from \doxylink{classsf_1_1_transformable}{sf\+::\+Transformable}\+: position, rotation, scale, origin. It also adds sprite-\/specific properties such as the texture to use, the part of it to display, and some convenience functions to change the overall color of the sprite, or to get its bounding rectangle.

\doxylink{classsf_1_1_sprite}{sf\+::\+Sprite} works in combination with the \doxylink{classsf_1_1_texture}{sf\+::\+Texture} class, which loads and provides the pixel data of a given texture.

The separation of \doxylink{classsf_1_1_sprite}{sf\+::\+Sprite} and \doxylink{classsf_1_1_texture}{sf\+::\+Texture} allows more flexibility and better performances\+: indeed a \doxylink{classsf_1_1_texture}{sf\+::\+Texture} is a heavy resource, and any operation on it is slow (often too slow for real-\/time applications). On the other side, a \doxylink{classsf_1_1_sprite}{sf\+::\+Sprite} is a lightweight object which can use the pixel data of a \doxylink{classsf_1_1_texture}{sf\+::\+Texture} and draw it with its own transformation/color/blending attributes.

It is important to note that the \doxylink{classsf_1_1_sprite}{sf\+::\+Sprite} instance doesn\textquotesingle{}t copy the texture that it uses, it only keeps a reference to it. Thus, a \doxylink{classsf_1_1_texture}{sf\+::\+Texture} must not be destroyed while it is used by a \doxylink{classsf_1_1_sprite}{sf\+::\+Sprite} (i.\+e. never write a function that uses a local \doxylink{classsf_1_1_texture}{sf\+::\+Texture} instance for creating a sprite).

See also the note on coordinates and undistorted rendering in \doxylink{classsf_1_1_transformable}{sf\+::\+Transformable}.

Change the source texture of the sprite. 

The {\itshape texture} argument refers to a texture that must exist as long as the sprite uses it. Indeed, the sprite doesn\textquotesingle{}t store its own copy of the texture, but rather keeps a pointer to the one that you passed to this function. If the source texture is destroyed and the sprite tries to use it, the behavior is undefined. If {\itshape reset\+Rect} is true, the Texture\+Rect property of the sprite is automatically adjusted to the size of the new texture. If it is false, the texture rect is left unchanged.
